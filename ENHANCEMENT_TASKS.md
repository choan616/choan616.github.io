# 📖 Diary PWA - 애플리케이션 고도화 과제

이 문서는 앱의 핵심 기능이 안정화된 현시점에서, 사용자 경험을 한 단계 더 높이고 앱의 가치를 더하기 위한 향후 개발 과제를 정의하고 우선순위를 정하기 위해 작성되었습니다.

---

## 🗺️ 고도화 로드맵

### 1. 사용자 경험(UX) 고도화

사용자가 매일 앱을 사용하면서 즐거움을 느끼고, 더 편리하게 일기를 관리할 수 있도록 만드는 기능들입니다.

#### 1.1 다크 모드 (Dark Mode)
- [x] **과제**: 라이트/다크 모드 전환 기능 구현
- **내용**: 사용자의 시스템 설정을 자동으로 감지하거나, 설정 메뉴에 토글 버튼을 추가하여 테마를 전환합니다. 야간 사용 시 눈의 피로를 줄여주고, 디자인 선택권을 제공하여 사용자 만족도를 높입니다.
- **난이도**: 🟢 **낮음** (CSS 변수 기반이므로 비교적 쉽게 구현 가능)
- **우선순위**: ⭐⭐⭐⭐⭐ (가장 먼저 구현 추천)

#### 1.2 태그 필터링 및 관리
- [x] **과제**: 태그를 활용한 일기 필터링 기능 구현
- **내용**: 사이드바 또는 별도 페이지에 사용된 모든 태그를 모아 보여주는 '태그 클라우드'를 만듭니다. 특정 태그를 클릭하면 해당 태그가 포함된 일기만 검색 결과처럼 보여줍니다.
- **난이도**: 🟢 **낮음** (기존 검색 로직을 활용하여 구현 가능)
- **우선순위**: ⭐⭐⭐⭐

#### 1.3 월별/연도별 통계
- [x] **과제**: 일기 작성 습관을 시각화하는 통계 대시보드 구현
- **내용**: 월별 일기 작성 개수, 가장 많이 사용한 태그, 일기 내용의 평균 길이 등을 차트나 그래프로 보여주어 사용자가 자신의 기록을 돌아보는 재미를 더합니다.
- **난이도**: 🟡 **중간** (데이터 집계 로직 및 `Chart.js`와 같은 차트 라이브러리 연동 필요)
- **우선순위**: ⭐⭐⭐

### 2. 핵심 기능 강화

일기 작성이라는 핵심 경험을 더 풍부하게 만들어주는 기능들입니다.

#### 2.1 마크다운(Markdown) 에디터 지원
- [ ] **과제**: 일기 작성 시 마크다운 문법 지원 및 미리보기 기능 추가
- **내용**: 사용자가 `#`으로 제목을, `*`로 목록을 만드는 등 간단한 마크다운 문법으로 일기 내용을 꾸밀 수 있게 합니다. 편집기 하단에 '미리보기(Preview)' 탭을 추가하여 실시간으로 결과물을 확인할 수 있습니다.
- **난이도**: 🟡 **중간** (`react-markdown` 라이브러리 등을 사용하여 구현)
- **우선순위**: ⭐⭐⭐⭐

#### 2.2 음성 녹음 및 위치 정보 추가
- [ ] **과제**: 일기에 음성 녹음 파일 및 위치 정보 첨부 기능 구현
- **내용**: 텍스트와 이미지뿐만 아니라, 음성 녹음이나 현재 위치(지도)를 함께 첨부하여 그날의 생생한 감정이나 장소를 기록합니다.
- **난이도**: 🟡 **중간** (브라우저의 `MediaRecorder`, `Geolocation` API 활용 필요)
- **우선순위**: ⭐⭐

### 3. 기술적 안정성 및 확장성 확보

장기적으로 앱을 더 견고하고 확장 가능하게 만드는 중요한 개선 작업들입니다.

#### 3.1 백업 버전 관리 시스템
- [ ] **과제**: 여러 버전의 백업 스냅샷을 보관하고, 특정 시점으로 복원하는 기능 구현
- **내용**: 현재의 덮어쓰기 방식은 데이터 유실 위험이 있으므로, 여러 버전의 백업을 관리하는 시스템을 도입합니다.
  - **파일 이름 규칙 변경**: `diary_backup.zip` 대신 `diary_backup_{타임스탬프}.zip` 형식으로 백업 파일을 생성합니다.
  - **백업 목록 표시**: 백업 패널에서 여러 버전의 백업 목록(생성 시간, 파일 크기 등)을 최신순으로 보여줍니다.
  - **버전별 복원**: 사용자가 목록에서 특정 버전을 선택하여 해당 시점의 데이터로 복원할 수 있게 합니다.
  - **저장 공간 관리**: 사용자의 저장 공간이 불필요하게 소모되는 것을 막기 위해, 오래된 백업을 자동으로 삭제하는 정책(예: 최근 20개만 유지)을 도입하고, 사용자가 수동으로 버전을 삭제할 수 있는 기능을 제공합니다.
- **난이도**: 🔴 **높음** (백업 파일 관리 및 복원 로직의 복잡성 증가)
- **우선순위**: ⭐⭐⭐

#### 3.2 증분 동기화(Delta Sync) 도입 검토
- [ ] **과제**: 변경된 데이터만 동기화하는 방식으로 동기화 효율 개선
- **내용**: 현재의 전체 데이터(스냅샷) 동기화 방식은 데이터가 많아지면 비효율적일 수 있습니다. 변경된 부분만 동기화하는 '증분 동기화' 방식을 도입하여 동기화 속도와 네트워크 사용량을 최적화합니다.
- **참고**: 이미지 데이터의 변경 추적 및 병합 로직이 매우 복잡하므로, 장기적인 과제로 신중하게 접근해야 합니다.
- **난이도**: 🔴 **매우 높음** (앱의 동기화 아키텍처를 근본적으로 변경해야 함)
- **우선순위**: ⭐

#### 3.3 다중 저장소 백엔드 지원 (ipdisk 등)
- [ ] **과제**: Google Drive 외에 Dropbox를 새로운 백업 저장소로 지원
- **내용**: 사용자가 Google Drive 또는 Dropbox를 백업 위치로 선택할 수 있도록 저장소 서비스 로직을 추상화하고 확장합니다. Dropbox는 공식 웹 API와 OAuth 인증을 지원하여 PWA 환경에 통합하기 용이합니다.
- **참고**: 신규 OAuth 2.0 인증 흐름 및 Dropbox API 연동 로직 구현이 필요합니다.
- **난이도**: 🔴 **높음** (신규 API 및 인증 로직 연동의 복잡성)
- **우선순위**: ⭐⭐

이제 앱의 안정성을 한 단계 더 높일 때입니다. **3.1 백업 버전 관리 시스템** 구현을 다음 단계로 강력히 추천합니다. 이 기능은 사용자의 소중한 데이터가 실수로 덮어씌워지는 것을 방지하는 핵심적인 안전장치입니다. 새로운 기능을 추가하는 것보다, 데이터 안정성을 확보하는 것이 앱의 신뢰도를 근본적으로 향상시키는 더 중요한 과제입니다.
